Compiling

GNU/Linux

There is a script installed at /usr/local/bin that will give you automatically the compiler and linker options that are needed to compile and link with the library.
Compiler flags: 3dengfx-config --cflags
Linker options: 3dengfx-config --libs

MS Windows/Visual Studio

  • Make sure you have MS Visual Studio .NET 2003 or newer, the old VS.NET compiler won't be able to compile the code, and it goes without saying that Visual C++ 6.0 won't cut it as well.
  • Use the included project to build the static library
  • Link with the static library 3dengfx.lib
  • Make sure you link with the Multithreaded DLL version of the C runtime
  • Add the src directory of this library at the include search path
  • Kill a chicken with a sacrificial blade over the roaring flames of an active volcano
  • Press build...

MS Windows/mingw (DevCpp)

DevCpp is a free IDE that comes with mingw , a port of gcc for windows. It also comes with a handy package manager that allows for easy installation of libraries.

  1. Download DevCpp and install.
  2. Go to this page and find the following packages:
    • zlib
    • libpng
    • cg
  3. Install them with a double click.
  4. Go here and get the latest SDL binaries for mingw.Copy the files libSDL.a and libSDLmain.a to the library folder of DevCpp (usually C:\Program Files\Dev-Cpp\lib). Rename the "include" folder of the package to "SDL" and copy it to the include directory of DevCpp (usually C:\Program Files\Dev-Cpp\include).
  5. create a new folder for the output files at the directory where you extracted this package. Name it "mingw_bin".
  6. Double click 3dengfx_devcpp.dev
  7. Go to project/project options/directories/include directories and add the directory where you installed 3dengfx. If you installed 3dengfx in C:\3dengfx , then add to the path "C:\3dengfx\src"
  8. Execute/compile. After this you should see in the mingw_bin folder among other intermediate files a file named "lib3dengfx.a" ...
